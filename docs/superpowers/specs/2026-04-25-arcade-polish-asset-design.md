# BalloonShoot v2 Arcade Polish Asset Design

作成日: 2026-04-25

## 位置づけ

この文書は `BalloonShoot_v2` の初回 polish pass における、ビジュアル方向、生成アセット、クロスヘア、射撃/命中演出、音の扱いを固定するための設計メモである。

この polish pass はゲームルール、2カメラ入力、手追跡、スコア計算を変更しない。目的は、既存 PoC の入力検証を保ったまま、見た目と音の手触りを「粗雑な仮UI」から「統一された明るいアーケード表現」へ寄せることである。

## 決定事項

- 方向性は **Arcade Celebration** とする。
- ただし、粗雑なテンプレート風 UI、過剰な装飾、絵文字、汎用アイコンセット風の見た目は採用しない。
- 風船とUI装飾は画像生成スキルで作成する。
- 生成アセットは同一パレット、同一線幅、同一影ルールで揃えたセットとして扱う。
- クロスヘアは画像生成せず、Canvas で動的に描画する。
- 射撃/命中演出は Canvas 側で制御し、画像素材に依存しすぎない。

## ビジュアル方針

### 基本トーン

子ども向けに明るく、ゲームらしい達成感を出す。放課後等デイサービスで使う PoC なので、刺激は強すぎず、読みやすさと安心感を残す。

目指す質感は「太いインク線」「硬い影」「ステッカー風」「明るいアーケード」である。ガラス風、ネオン過多、紫ピンクの汎用グラデーション、過剰なぼかし影は避ける。

### パレット

初回 polish pass では次のパレットを基準にする。

- `ink`: `#1a1430`
- `cream`: `#fff5e1`
- `candy`: `#ff5a8a`
- `lime`: `#c8ff3a`
- `sky`: `#40c7ff`
- `mint`: `#00d1b2`
- `alert`: `#ffb400`

生成アセット、HUD、結果UI、Canvas演出はこの範囲を基準にする。多色のランダムな紙吹雪や虹色表現は使わない。

## 生成アセット

画像生成スキルで作るアセットは、単発の装飾ではなく、同じ画面に並べても破綻しない統一セットとして生成する。

### 作成対象

1. 風船スプライト
   - 通常サイズ用のメイン風船
   - 小型風船用の小さめ風船
   - 色違いのバリエーション
   - 太い `ink` アウトライン、硬い影、控えめなハイライト
2. 結果画面用スター評価バッジ
   - 1〜3個表示に使える、同一形状のステッカー風スター
   - `alert` と `cream` を中心に、`ink` アウトラインで視認性を確保
3. ヒットスパーク素材
   - 命中時の短いアクセントに使える小型素材
   - Canvas のリング/破片演出を補助する目的に限定
4. リボン紙吹雪素材
   - 点ではなく短いリボン形状
   - 結果画面や高コンボ時の控えめな演出に使う
5. 結果画面用小物
   - CTA周辺やスター周辺に置ける小さなステッカー素材
   - 絵文字や既製アイコンに見えない、同一パレットの生成素材

### 保存方針

- source 側は `src/assets/images/` に置く。
- runtime で参照する最終PNGは `public/images/` に置く。
- 透明背景が必要なアセットは、画像生成後に背景除去して PNG alpha を確認する。
- 既存ファイルは明示的な置換判断なしに上書きしない。

## 禁止する表現

- 絵文字を UI や結果画面やアセットとして使わない。
- Material Icons、Heroicons、汎用Webアイコン風の見た目にしない。
- ランダムな虹色パーティクルを使わない。
- ぼかしの強いソフトシャドウを多用しない。
- ガラス風カード、過剰な半透明ぼかしパネルを使わない。
- 英語だけの汎用結果コピーを主役にしない。
- スコアを単なる `Score: 0` のような仮表示にしない。

## クロスヘア

クロスヘアは画像生成アセットではなく、Canvas で描画する。

### 形状

- 外周リングを持つ。
- リング外周は `ink` で支える。
- リング本体と十字線は `cream` を基準にする。
- 十字線はリングの中に収める。
- 十字線の端はリング内側に触れる程度にする。
- 十字線にはアウトラインを付けない。
- 中央ドットは置かない。

### 理由

中央に塗りやドットがあると、風船の境界や実際の狙い位置を隠す。十字線がリング外にはみ出すと、見た目のノイズが増え、アーケード風の整理された印象から外れる。

## 射撃/命中アニメーション

### 射撃時

- クロスヘア全体を短時間だけ縮ませる。
- 目安は 120ms 程度で `1.0 -> 0.72 -> 1.0`。
- 画面全体のフラッシュは使わない。
- 中央ドットや追加アイコンは出さない。

### 命中時

- クロスヘアは短く色変化する程度に抑える。
- 命中の主役は別のヒットリング、風船破片、浮遊スコア表示にする。
- ヒットリングは `ink` の輪郭を持ち、短時間で外へ広がって消える。
- 破片は 5〜7個程度の大きめポリゴンまたはリボン形状にする。
- 浮遊スコアは `+1` や `+3` を短時間表示し、上方向へ少し動かして消す。
- 高コンボ時以外は画面揺れを使わない。

## HUD

HUD は現在の情報量を保ちながら、仮UI感を減らす。

- スコアは左上のステッカー風バッジにする。
- タイマーは読みやすい円形または太いバッジにする。
- コンボと倍率は必要な時に目立つチップとして出す。
- 数字は大きく、視線移動が少ない配置にする。
- カメラ背景上でも読めるよう、`ink` アウトラインと硬い影を使う。
- 画面端に詰めすぎず、余白を確保する。

## 結果画面

結果画面は達成感を出すが、情報量を増やしすぎない。

- 見出しは短い日本語にする。
- 最終スコアを最も大きく見せる。
- 最大コンボはサブ情報として扱う。
- スター評価バッジを1〜3個表示できる余地を持たせる。
- CTA は `もういっかい` を主ボタンにする。
- 統計テーブルやランキングや共有導線は入れない。

## 音

既存の `AudioController` の責務は保つ。

- 射撃音は銃声に寄せない。短いポップ音、クリック音、コルク音に寄せる。
- 命中音はポップ音と軽いディング音を重ねる方向にする。
- タイムアップ音は警報ではなく、短い終了フレーズにする。
- 結果音は短い祝福ジングルにする。
- BGM はゲーム中に邪魔にならない音量に抑える。
- 音量や将来の ducking は named token として扱い、マジックナンバーを避ける。

## 実装境界

- `src/features/rendering/`: クロスヘア、ヒットリング、破片、浮遊スコアを描画する。
- `src/app/balloonGameRuntime.ts`: 既存の射撃/命中イベントから演出を発火する。
- `src/app/gameHud.ts`: HUDと結果UIのHTML構造を整理する。
- `src/styles/app.css`: HUD、結果UI、ボタン、バッジの見た目を定義する。
- `src/features/audio/createAudioController.ts`: 音量、差し替え、将来の ducking を扱う。
- `src/assets/images/` と `public/images/`: 生成画像アセットを整理する。

ゲームルール、スコア計算、入力融合、カメラライフサイクルはこの polish pass の対象外とする。

## 初回実装スライス

初回は次の順で小さく進める。

1. 画像生成する風船/アイコンセットのプロンプトを固定する。
2. 風船スプライトとUI装飾素材を生成し、source/runtime の置き場を整理する。
3. クロスヘア描画を最終形に変更する。
4. 射撃縮みと命中リング/浮遊スコア/破片を追加する。
5. HUDをステッカー風に整える。
6. 結果画面をスコア主役の構成にする。
7. 音声差し替えまたは音量整理を行う。

## 受け入れ条件

- UIに絵文字が出ない。
- 生成アセットの線幅、影、パレットが揃っている。
- クロスヘア中央が空いている。
- 十字線がリング外にはみ出していない。
- 射撃時にクロスヘアが短く縮む。
- 命中時にヒットリング、破片、浮遊スコアのいずれかが見える。
- HUDの主要数字がカメラ背景上で読める。
- 結果画面で最終スコアが主役になっている。
- 既存の `lint`, `typecheck`, `test` が通る。
